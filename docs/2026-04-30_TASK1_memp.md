## What MemP is (vs Dynamic Cheatsheet)

| | Dynamic Cheatsheet (M1-u) | MemP |
|---|---|---|
| Granularity | one monolithic primer | one entry per train trajectory |
| Distillation | aggregated across 18 train tasks | per-task workflow paragraph |
| Retrieval | none (always inject) | cosine sim on task descriptions |
| At test time | constant 820-token primer | top-k retrieved entries (~1500 tok @ k=3) |
| Mechanism | aggregated rules + element-name tables | task-conditioned procedural narratives |

For our setup:
- **Build**: each of 18 train trajectories distilled to a workflow
  paragraph via gemini-3-flash-preview using MemP's "direct" build prompt.
  Library at `misc/memory_artifacts/memp_dsv4/library.json` (18 entries +
  qwen3-embedding-8b embeddings of task descriptions).
- **Retrieve**: at test time, compute cosine similarity between the
  current test task's instructions and each train task's instructions.
  Take top-3.
- **Render**: concatenate the 3 retrieved workflows into a per-task
  primer at `plugin/memp_per_task/<test_task>.md`.
- **Inject**: runner now supports `cheatsheet_path_template` with `{task}`
  substitution. `abl_cMP_a_memp_on_c2` and `abl_cMP_b_memp_on_c7` use it.

## Hypothesis (pre-experiment)

MemP should win where individual train tasks share strong physics-class
signal with test tasks (e.g., poroelastic-train → poroelastic-test).
DC-style memory wins where failure modes are diffuse across tasks.

For GEOS XML, many task families are clearly clustered (wellbore,
fracture, poromechanics, tutorial-style). MemP retrieval should latch
onto right train memory.

## Caveat: distilled content quality

The 18 distilled workflows pass token / hygiene gates (no GT XML
filenames after stripping) but still contain **plausible-sounding GEOS
hallucinations** in some entries — e.g., `SolidMechanicsLagrangianSSLE`
(real one is `SolidMechanicsLagrangianFEM`), `BiotLinearPoromechanics`
(not a real solver name), `FractureManager`. The "ungrounded" M1-u
distillation had the same issue and was the canonical example of the
risk.

This is a known limitation of self-judged distillation without TreeSim
grounding — the distiller fabricates plausible names because nobody is
checking against the schema.

A "grounded" MemP variant that uses TreeSim feedback to guide
distillation could fix this, but: when the train trajectories are all
"success" (which our 18 are — mean treesim 0.895 under C2 setup), there
are no failure modes to ground on.
